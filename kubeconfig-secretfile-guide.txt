# Option B: Use Jenkins Credentials (Secure)

Go to Jenkins UI → Manage Jenkins → Credentials.
Add a Secret File, upload your kubeconfig, and give it an ID (e.g., kubeconfig).

# In your Jenkinsfile:

withCredentials([file(credentialsId: 'kubeconfig', variable: 'KUBECONFIG')]) {
    sh 'kubectl get pods'
    sh 'kubectl apply -f manifests/deploy.yaml'
}

Jenkins will:
Inject the kubeconfig as a temp file.
Set the KUBECONFIG env var to that temp file.

# Ensure Network Access to Kubernetes

The agent must be able to reach your cluster’s API server.
If the cluster is on Minikube or a local dev cluster, you may need to configure the correct IP and certificate paths in the kubeconfig.
For cloud clusters (EKS, GKE, AKS), ensure the agent can reach public/private endpoints as required.

Step 1: Add kubeconfig as Secret File in Jenkins

Go to Jenkins Dashboard → Manage Jenkins → Manage Credentials.
Select the relevant domain (usually Global).
Click Add Credentials.
For Kind, choose Secret file.
Upload your kubeconfig file.
Give it an ID (e.g., kubeconfig-secret).
Save.

Step 2: Use kubeconfig in your Jenkinsfile

You can wrap your kubectl commands inside a withCredentials block that injects the kubeconfig file as an environment variable or file.
Example:

pipeline {
    agent { label 'jenkins-s1' }

    stages {
        stage('Deploy to Kubernetes') {
            steps {
                withCredentials([file(credentialsId: 'kubeconfig-secret', variable: 'KUBECONFIG')]) {
                    sh '''
                        echo "Using kubeconfig from secret file"
                        kubectl --kubeconfig=$KUBECONFIG apply -f manifests/deploy-simple-microservice.yaml
                    '''
                }
            }
        }
    }
}

What happens here?

Jenkins downloads the secret kubeconfig file and assigns its path to environment variable KUBECONFIG.
kubectl uses this kubeconfig to authenticate and communicate with your Kubernetes cluster.
